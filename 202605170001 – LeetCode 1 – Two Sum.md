# LeetCode 1 – Two Sum

## Задача

Дан массив чисел и число `target`. Найти два индекса, чьи элементы в сумме дают `target`. Вернуть пару индексов. Решение единственное.

## Первая реакция

Первая мысль — два цикла, брут форс O(n²). Но это очевидно медленно. Нужно что-то, что позволяет искать «дополнение» за O(1) — значит, hash map.

## Подходы

### ❌ Подход 1 — Brute force

**Идея:** для каждого элемента перебрать все оставшиеся и проверить сумму.

**Почему не работает / слишком медленно:** O(n²) — при n=10⁴ это 10⁸ операций.

---

### ✅ Финальный подход — Hash Map за один проход

**Ключевой инсайт (одна фраза):** храни в map не сам элемент, а его «дополнение» — тогда при следующем шаге сразу видно, нужен ли этот элемент.

**Алгоритм по шагам:**
1. Идём по массиву с индексом `i`.
2. Проверяем: есть ли `nums[i]` в нашей map?
3. Если да — нашли пару (возвращаем текущий индекс + сохранённый).
4. Если нет — кладём в map: `complement = target - nums[i]` → `map[complement] = i`.

**Код (ключевые фрагменты):**
```python
seen = {}
for i, num in enumerate(nums):
    if num in seen:
        return [seen[num], i]
    seen[target - num] = i
```

**Сложность:**
- Время: O(n)
- Память: O(n)

## Что было сложно

Поначалу хотел сохранять сам элемент как ключ, а не его дополнение — это приводило к двум проходам. Понял, что если сохранять `target - num`, то всё решается за один проход: элемент сам себя «ищет» при следующей итерации.
